# 08 回滚与应急处理

## 场景说明（我在做什么）
模拟盘或实盘表现突然恶化，需要立刻降风险或停机。

## 复制即用命令（Windows PowerShell）
```powershell
freqtrade trade --config user_data/configs/S5_config_stub.json --strategy S5_LFT_Conservative_MTF_TrendReversal --dry-run
```

> 应急时优先动作：把配置切回 `dry_run: true`、`max_open_trades` 降到 0 或 1，并只保留保守策略（S1/S3/S5）。

## 参数解释（大白话）
- 回滚不是“修好再说”，而是“先止血”。
- 先停新增仓，再处理旧仓风险。

## 结果怎么看（成功/失败判断）
- 成功：没有新开仓，风险暴露明显下降。
- 失败：如果仍在频繁下单，说明配置未生效或进程未重启。
